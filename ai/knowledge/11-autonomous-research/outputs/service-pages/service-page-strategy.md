## Conversion Architecture

### Conversion Points Per Page

Each service page should include 4-6 conversion opportunities:

1. **Hero CTA** (above fold): "Get Free Estimate" - Primary conversion
2. **Mid-page sticky** (after benefits): "Call Now: (907) 310-3000" - Phone preference
3. **Inline form** (after process section): Quote request form - Lead capture
4. **Social proof CTA** (after testimonials): "See Our Work" - Gallery/portfolio
5. **Exit intent** (on scroll/leave): Special offer popup - Recovery
6. **Footer CTA**: Contact form + phone + scheduling - Final capture

### Required Page Elements

Every service page MUST include:

1. **Hero Section**
   - Compelling headline with primary keyword
   - Subheadline addressing pain point
   - Hero image (local project photo preferred)
   - Primary CTA button
   - Trust indicators (3)

2. **Problem/Solution Section**
   - Address specific customer pain points
   - Introduce service as solution
   - Alaska climate relevance

3. **Service Explanation**
   - What the service involves
   - Technical details (accessible language)
   - Materials/products used

4. **Benefits Section**
   - 7-10 benefits with icons
   - Energy savings focus
   - Comfort improvements
   - Long-term value

5. **Process Section**
   - Step-by-step installation process
   - Timeline expectations
   - What customers should expect

6. **Pricing/ROI Section**
   - Transparent pricing guidance (ranges acceptable)
   - Energy savings estimates
   - ROI calculations
   - Available rebates/incentives

7. **FAQ Section**
   - 10-15 questions minimum
   - Schema markup required
   - Address objections

8. **Social Proof**
   - Customer testimonials (2-3)
   - Before/after photos
   - Case study snippet

9. **Related Services**
   - 3-4 related services with links
   - Cross-selling opportunities

10. **Final CTA Section**
    - Strong closing headline
    - Contact form
    - Phone number prominent
    - Address with map

## Technical Requirements

### Schema Markup

Every service page must have:

```json
{
  "@context": "https://schema.org",
  "@type": "Service",
  "serviceType": "[Service Name]",
  "provider": {
    "@type": "LocalBusiness",
    "name": "Foamology Insulation",
    "address": {
      "@type": "PostalAddress",
      "streetAddress": "901 E Klatt Road #6",
      "addressLocality": "Anchorage",
      "addressRegion": "AK",
      "postalCode": "99515"
    },
    "telephone": "(907) 310-3000",
    "areaServed": ["Anchorage", "Mat-Su Valley", "Kenai Peninsula Borough"]
  },
  "description": "[Service description]"
}
```

Plus FAQPage schema for FAQ sections.

### Page Speed Requirements

- **LCP**: < 2.5 seconds
- **FID**: < 100ms
- **CLS**: < 0.1
- Image optimization: WebP with JPEG fallback
- Lazy loading for below-fold images

### Mobile Optimization

- Touch-friendly CTAs (minimum 44x44px)
- Click-to-call phone numbers
- Collapsible FAQ accordions
- Responsive image sizing
- Fast mobile load times
